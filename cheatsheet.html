Skedulo CLI Cheatsheet

The Skedulo CLI is a modern, powerful tool designed to streamline workflows, enable source-code driven development, and simplify building, testing, and deploying custom solutions on the Skedulo platform.

Installation

  • Installation Guide
  • About: $ sked about
  • Available versions: $ sked update --available
  • Upgrade latest version: $ sked update

Internal Plugin

  • Install: $ sked plugins install @skedulo/plugin-internal
  • Login: $ sked internal login [-e <test/production>]
  • More: $ sked internal --help

Internal SDLC

$ sked package deploy local -p </package/dir> -a <alias>
Not allowed to deploy to a Production environment.

$ sked internal register-package -s <github-reference-url>
Register a Skedulo package. The package name must start with skedulo-core/ or skedulo-extension/ or skedulo-solution/.

$ sked internal deploy-package -p package-name -t <team-name> [-v <version>]
Deploy a registered Skedulo package. The package name must start with skedulo-core/ or skedulo-extension/.

Customer SDLC

$ sked package deploy local -p </package/dir> -a <alias>
Not allowed to deploy to a Production environment.

$ sked tenant register -s <github-reference-url>
Register a customer package. The package name must start with the team name of the Production tenant of the customer.

$ sked package deploy registered -p <package-name> [-v <version>]
Deploy a registered customer package.
